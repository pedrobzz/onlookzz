# Electron to Web Migration

Onlook's current web architecture keeps the visual editor in a browser while preserving local filesystem and process access through a small local runtime.

## Mapping

```txt
BrowserView -> Next.js app
WebView     -> iframe preview with preload bridge
Node server -> local Elysia runtime
Local JSON  -> Convex metadata plus local project folders
```

## Frontend

The editor UI moved into the Next.js app. Client/server boundaries follow App Router conventions, and app metadata is read and written through Convex.

## Canvas and Frame

The preview runs in an iframe. Onlook injects a preload script into the preview app so the editor can map DOM elements to source code and apply visual edits.

## Runtime

The local runtime replaces Electron's direct Node access. It owns filesystem operations, watchers, install/dev/command processes, logs, preview ports, and SSE events.

Project source files stay on disk under `.onlook/sandboxes/<project-id>`. Convex stores metadata and realtime app state, not source file contents.

## Removed Web-era Services

The standalone app no longer uses Supabase, tRPC, CodeSandbox, app-level branches, SaaS auth, billing, teams, custom domains, or Docker deployment.
