# Self-Hosting

Onlook is designed to run as a standalone local app backed by Convex and a local runtime. It does not require Supabase, Docker Compose, OAuth setup, billing infrastructure, or CodeSandbox.

## What You Host

- The Next.js web client.
- Convex for app metadata and realtime state.
- The local runtime on the same machine as the project folders.
- Project source files under `.onlook/sandboxes/<project-id>`.

## What Stays Local

The runtime must have direct access to the filesystem and process manager. It owns file reads and writes, watchers, installs, dev processes, one-shot commands, logs, preview ports, and runtime SSE events.

## Guides

- [Single Machine](/self-hosting/single-machine): run the standalone app and local runtime on one machine.
- [External Services](/self-hosting/external-services): configure model/search providers used by AI features.
