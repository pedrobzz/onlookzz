# Single Machine

This setup runs the web app, Convex development connection, local runtime, and project folders on one machine.

## Prerequisites

- Bun
- A Convex development deployment
- Model/API keys for enabled AI providers

## Install

```bash
git clone https://github.com/onlook-dev/onlook.git
cd onlook
bun install
```

## Configure

Create `apps/web/client/.env.local`:

```bash
NEXT_PUBLIC_CONVEX_URL=
MORPH_API_KEY=
RELACE_API_KEY=
FIRECRAWL_API_KEY=
EXA_API_KEY=
```

## Run

Start Convex:

```bash
bunx convex dev
```

Start the app from the repository root:

```bash
bun dev
```

## Project Data

Project metadata lives in Convex. Project source files live on disk:

```txt
.onlook/sandboxes/<project-id>
```

Back up both Convex data and the `.onlook/sandboxes` directory if you need to preserve local projects.

## Runtime Notes

The local runtime is required for filesystem access, process execution, logs, watcher events, and preview ports. If previews or file writes fail, verify that the runtime can access the project folder and that the project id matches the Convex project.
