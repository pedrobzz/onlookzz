# Running Locally

Onlook is a standalone local app. The web client talks to Convex for app metadata and to a local runtime for project files, commands, logs, and previews.

## Prerequisites

- [Bun](https://bun.sh/)
- A Convex development deployment
- Any model/API keys required by the AI features you want to use

Docker, Supabase, CodeSandbox, and database migration commands are not part of the local development path.

## Install Dependencies

```bash
bun install
```

## Configure Environment

Create `apps/web/client/.env.local` and provide the values validated by `apps/web/client/src/env.ts`.

```bash
NEXT_PUBLIC_CONVEX_URL=
MORPH_API_KEY=
RELACE_API_KEY=
FIRECRAWL_API_KEY=
EXA_API_KEY=
```

Only `NEXT_PUBLIC_CONVEX_URL` is exposed to the browser. Server-only keys must not be imported into client code.

## Start Convex

Run Convex with the app's Convex directory:

```bash
bunx convex dev
```

## Run Onlook

From the repository root:

```bash
bun dev
```

The app opens to the project list. Project source files are prepared under `.onlook/sandboxes/<project-id>` and runtime operations flow through the local runtime.

## Validate Changes

Use focused checks while developing:

```bash
bun --filter @onlook/web-client typecheck
bun test
```
