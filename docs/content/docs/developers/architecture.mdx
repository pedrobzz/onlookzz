# Architecture

Onlook is a Bun monorepo centered on a standalone Next.js app.

```txt
Next app
  -> Convex
       - projects
       - frames
       - conversations and messages
       - settings
       - checkpoints
       - uploads
       - realtime app subscriptions
  -> Local Elysia runtime
       - filesystem access
       - filesystem watcher
       - process manager
       - command execution
       - logs
       - preview ports
       - runtime SSE
  -> .onlook/sandboxes/<project-id>
```

## Repo Map

- `apps/web/client`: Next.js App Router app.
- `apps/web/client/convex`: Convex schema, queries, and mutations.
- `packages/file-system`: browser file cache and local runtime client/server pieces.
- `packages/ai`: prompts, tools, and model integration.
- `packages/models`: shared models for the current standalone app.
- `packages/ui`: shared UI components.

## Data Boundaries

Convex owns app metadata and realtime state. The local runtime owns anything that needs machine-local access: files, commands, process logs, preview ports, and watcher events.

Project source files stay on disk. The browser keeps an optimistic cache, but the local project folder is the source of truth for project source contents.

## Removed Concepts

The current architecture does not include Supabase, tRPC, CodeSandbox, SaaS auth, users, members, teams, billing, Stripe, custom domains, Docker deployment, or app-level branches.
